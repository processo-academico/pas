Role: Projetista
O Projetista define as responsabilidades, operações, atributos e relacionamentos de uma ou várias classes e determina como eles serão ajustados para o ambiente de implementação. É responsável, tembém, pelos pacotes ou subsistemas do projeto, incuindo todas as suas classes.
Relationships
Performs:
  • Escolher Casos de Uso para Validar a Arquitetura
  • Identificar Mudanças nos Requisitos
  • Iniciar Detalhamento do (s) Caso (s) de Uso Escolhido (s) para Validar Arquitetura
  • Realizar Análise e Projeto
Responsible for:
  • Especificação de Caso de Uso
Additionally Performs:
  • Consultar ou Entrevistar Cliente
Modifies:
  • Especificação de Caso de Uso

O Projetista deve ter conhecimentos práticos de:

  • Técnicas de modelagem de casos de uso
  • Requisitos do sistema
  • Técnicas de projeto de software, incluindo as técnicas de análise e projeto orientado a objetos e a Linguagem de Modelagem Unificada (UML)
  • Tecnologias com as quais o sistema será implementado

O Projetista deve também estar a par da arquitetura do sistema, através do Documento de Arquitetura do Sistema.
