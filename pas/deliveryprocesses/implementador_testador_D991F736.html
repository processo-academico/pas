Role: Implementador e Testador
O Implementador é responsável por desenvolver e testar componentes de acordo com os padrões adotados para o projeto, para fins de integração com subsistemas maiores. Esse papel também é responsável por desenvolver e testar os componentes e os subsistemas correspondentes.
Relationships
Performs:
  • Executar Builds Necessários, Integrar (caso necessário) e criar um Release
  • Executar Implantação
  • Executar Teste Beta e Coletar Informações
  • Executar Testes Alfa em Build Estável no Ambiente de Desenvolvimento
  • Gerar Relatório com Dados Coletados
  • Implementar Correção de Erros
  • Planejar Testes Alfa em Ambiente de Desenvolvimento
  • Verificar se Necessita passar por Implementação ou se deve passar por Remodelagem
Responsible for:
  • Códigos Fontes
  • Plano de Testes
Additionally Performs:Modifies:
  • Códigos Fontes
  • Plano de Testes
Main Description

De uma forma geral, o Implementador tem a função de implementar e testar componentes de um subsistema isolado. Em processos prescritivos como o RUP, as funções de implementação, integração e testes são desempenhadas por papéis separados. A proposta para o PAS, porém, é que o Implementador acumule funções de implementação, integração e testes do sistema.

Dessa forma, após implementar os componentes, o Implementador, no PAS, combina os componentes para criar um build. Além disso, esse papel deve conduzir os testes necessários e registrar seus resultados.

As habilidades e conhecimentos apropriados para o implementador incluem:

  • Conhecimento do sistema no qual está trabalhando
  • Habilidades de programação
  • Conhecimento da linguagem de programação utilizada
  • Familiaridade com as ferramentas de programação e de testes
